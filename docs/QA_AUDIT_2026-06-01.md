## 🔴 CRITICAL — Fixed this session

### C1. Undefined token `bg-bg-overlay` → invisible modal headers
**Files:** `FileUpload.tsx:46`, `DocumentVault.tsx:50`
The file-preview modal header used `bg-bg-overlay`, a token never defined in
`index.css`. In Tailwind 4, undefined color utilities silently render nothing — so the
modal header bar (file name + download + close) floated with **no background** over the
blurred backdrop.
**Fix:** Added `--color-bg-overlay: #141417` to the theme.

### C2. Undefined token `bg-bg-card` → vault rows had no background
**File:** `DocumentVault.tsx:168`
Document Vault list rows used `bg-bg-card` (undefined) → rows rendered with a border but
no fill, breaking the card visual.
**Fix:** Added `--color-bg-card: #141417`.

### C3. Undefined token `border-border-hover` → dead hover state
**File:** `DocumentVault.tsx:168`
`hover:border-border-hover` referenced an undefined token → no hover feedback on rows.
**Fix:** Added `--color-border-hover: #44444C`.

> **Root cause / process note:** these three slipped in because new components were
> written against assumed tokens without cross-checking `index.css`. **Recommendation:**
> the design system is small (one `@theme` block) — keep a tokens cheat-sheet, or add a
> lint rule. UI Polish phase should add a Storybook-style token reference page.

## 🟠 HIGH

### H1. Mobile navigation shell broken — ✅ Fixed this session
**Files:** `AppShell.tsx`, `Sidebar.tsx`
`ProtectedRoute` lays out `<Navbar/>` + `<main>` in a horizontal `flex` row. The Navbar
returned the mobile top-bar and the sidebar `<aside>` as **flex siblings in that row**.
Result on a phone:
- The mobile header rendered as a squished narrow **left column**, not a full-width top bar.
- Opening the menu **pushed content sideways** (w-60 flex child) instead of overlaying.

The app was effectively unusable on mobile.
**Fix:** Mobile top bar → `fixed top-0 inset-x-0 z-40` (full-width). Sidebar → `fixed
inset-y-0 left-0 z-50` overlay on mobile with a `bg-black/50` backdrop; stays
`md:relative md:block` on desktop. `<main>` gets `pt-12 md:pt-0` to clear the fixed bar.

> This fixes *navigation*. Per-page responsive layout (tables, multi-column grids,
> forms) remains the **Mobile Responsive** phase — see M-series below.

### H2. Per-page responsive layout gaps → Mobile Responsive phase
Several pages assume desktop width. Worst offenders found:
- **`PurchaseRequestDetail.tsx`** — sourcing matrix + price-comparison are wide tables.
  They use `overflow-x-auto` (good, won't break), but on a phone the horizontal scroll +
  number inputs are awkward. Needs a card/stacked alternative on small screens.
- **`WorkOrderDetail.tsx`** — `InvoiceForm` uses a 12-column grid (`grid-cols-12`) for
  line items; collapses poorly under ~400px.
- **`PurchaseOrderDetail.tsx`** — payment-chain + line items are fine, but the 3-column
  outer grid stacks acceptably. Verify spacing.
- Most dashboards already use `grid-cols-3 md:grid-cols-5` etc. and are OK.

**Proposed:** mobile-first pass per page; tables → card lists under `md`; forms stack to
single column; consider sidebar → bottom-tab nav for primary routes on mobile.

## 🟡 MEDIUM — code quality (→ Polish phase)

### M1. `attachments` field missing from entity types → `as any` casts
**Files:** `TicketDetail`, `PurchaseOrderDetail`, `FuelRequestDetail`, `WorkOrderDetail`
The new `Attachment[]` lives only in `storage.ts`. Entity interfaces
(`workflow-entities.ts`, `crm.ts`) were never updated, so every page reads
`(po as any).attachments`. This bypasses the type checker.
**Proposed:** Move `Attachment` to a shared types module; add `attachments?: Attachment[]`
to `Ticket`, `PurchaseOrder`, `FuelRequest`, `WorkOrder`. Remove all 4 casts.

### M2. Two competing document models
`workflow-entities.ts` still defines `documents: UploadedDoc[]` on Ticket/PO/FuelRequest,
which predates the new `attachments` + Document Vault system. They now coexist, unused vs
used. **Proposed:** Deprecate `UploadedDoc`/`documents`, standardise on `Attachment`.

### M3. Procurement quotes hardcode MVR
**File:** `PurchaseRequestDetail.tsx:83`, labels at `:211`, `:222`
Quote build hardcodes `currency: 'MVR'` and the matrix labels say "(MVR)", ignoring the
money foundation (`money.ts` / `CurrencySelector`) used everywhere else in finance.
**Proposed:** Thread the PR/PO currency through the sourcing + comparison grids.

### M4. `arrayRemove(attachment)` exact-match fragility
**File:** `storage.ts` `deleteEntityFile`
Firestore `arrayRemove` requires a byte-exact object match. If a stored attachment ever
differs from the object passed in (e.g. an older record without `vaultDocId`, or field
order drift), the Storage object is deleted but the **array metadata is orphaned**.
**Proposed:** Delete by `storagePath` match (read array, filter, write back) instead of
relying on object-identity `arrayRemove`.
